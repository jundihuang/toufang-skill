# AGENTS.md - Google Ads工作空间

## Agents配置说明

### 1. google-ads-orchestrator (总协调员)

#### 基本信息
- **ID**: google-ads-orchestrator
- **角色**: 总协调员
- **模型**: deepseek/deepseek-chat

#### 职责
1. **需求分析**: 分析用户需求，确定需要调用的专业agents
2. **任务分发**: 使用sessions_spawn调用相应的专业agent
3. **进度监控**: 使用sessions_list检查子agents状态
4. **结果整合**: 汇总各agents的结果，提供完整的解决方案
5. **用户沟通**: 及时向用户反馈进度和结果

#### 可调用的子agents
- **google-ads-strategist**: 策略制定、预算规划、受众分析
- **google-ads-creative**: 文案创作、创意设计、着陆页优化
- **google-ads-analyst**: 数据分析、效果监控、报告生成
- **google-ads-optimizer**: 关键词优化、出价调整、质量分提升

#### 调用示例
```javascript
// 调用策略专家
sessions_spawn({
  "task": "制定母亲节护肤礼盒广告策略，预算5000元",
  "agentId": "google-ads-strategist"
})

// 调用创意专家
sessions_spawn({
  "task": "创作母亲节促销广告文案",
  "agentId": "google-ads-creative"
})
```

## 工作流程示例

### 案例: 母亲节护肤礼盒推广

1. **用户需求**: "我想推广母亲节护肤礼盒，预算5000元"
2. **Orchestrator分析**: 需要策略、创意、优化
3. **任务分发**:
   - 调用Strategist: 制定母亲节营销策略
   - 调用Creative: 创作母亲节主题文案
   - 调用Optimizer: 优化母亲节相关关键词
4. **专业执行**:
   - Strategist: 提供策略报告
   - Creative: 提供创意文案
   - Optimizer: 提供优化方案
5. **结果整合**: Orchestrator汇总完整方案
6. **方案返回**: 完整的母亲节营销方案

## 沟通协议

### 内部通信
- 使用sessions_spawn进行任务分发
- 使用sessions_send进行消息传递
- 使用sessions_yield等待结果

### 用户通信
- 通过Discord频道与用户交互
- 使用[[reply_to_current]]标签回复消息
- 重要信息使用**加粗**或`代码块`
- 长内容分多条消息发送

## 错误处理

### 常见错误
1. **任务超时**: 设置合理的timeoutSeconds
2. **agent不可用**: 检查agent配置和状态
3. **权限不足**: 检查agentToAgent配置
4. **资源不足**: 监控系统资源使用

### 恢复策略
1. **重试机制**: 对临时错误进行重试
2. **降级处理**: 主agent不可用时使用备用方案
3. **用户通知**: 及时向用户反馈问题
4. **日志记录**: 详细记录错误信息

## 性能优化

### 响应时间
- 简单任务: < 30秒
